> **Note: Mailup is only available on the pro version.**

***To integrate with Mailup follow, the below steps.***


## Process of Authorization and Integration

1.	Search & Select **Mailup** from the actions pages
 <Image src="actions/mailup/search-and-select" alt="Mailup" w="1020" h="350" />

2. **Integration Name:** Set the proper integration name. Default it's taken 'Mailup'.
 <Image src="actions/mailup/default-integration-name" alt="Mailup" w="1020" h="350" />

3.	After that, you have to copy the client id and client secret from your Mailup account & paste the client id and client secret on the field.     
**To get Mailup client id and client secret**
-	First login your Mailup account
-	Go to setting > advanced setting > Developer's corner > API Keys
-	Create a **NEW API KEY** or copy the client id and client secret from the previously created API key
-	Then paste into client id and client secret field of your authorization form. And Click on **Authorize** button

 <Image src="actions/mailup/visit-api-console" alt="Mailup" w="1020" h="350" />
 <Image src="actions/mailup/authorize" alt="Mailup" w="1020" h="350" />


4. Click on **`Next`** to set up the other integration settings ahead.
 <Image src="actions/mailup/authorize-after" alt="Mailup" w="1020" h="350" />

5.	After completing the authorization process, Select the `List` and `groups` of your choices from the drop-down options. If you don't see any list and groups. Click on the `refresh` button.
 <Image src="actions/mailup/select-status" alt="Mailup" w="1020" h="350" />

6.	You will find the Map Field option. You have to map all the necessary fields with the related Mailup field. By default, **`Email`** field of Mailup will be available. You can also send custom values by mapping your required field. then, click on the **`Next`** button to move forward for further settings.

 <Image src="actions/mailup/mailup-field-map" alt="Mailup" w="1020" h="350" />

7. Now save the integration by clicking the **`Finish & Save`** Button

 <Image src="actions/mailup/finish-save" alt="Mailup" w="1020" h="350" />


## Actions 
- **Double Op-tin:** By enabling the Double-Opt-in action admin can ask permission from users if they want to subscribe to MailBluster or not.
 <Image src="actions/mailup/opt-in" alt="MailBluster" w="1020" h="350" />


8. We have an option in every integration **`Timeline`** where you can check integration success or error status.
 <Image src="actions/mailup/mailup-timeline" alt="Mailup" w="1020" h="350" />
 <Image src="actions/mailup/mailup-log" alt="Mailup" w="1020" h="350" />

9. Also, you can **`Edit`** **`Clone`** **`Delete`** the integration.

 <Image src="actions/mailup/mailup-option" alt="Mailup" w="1020" h="350" />
