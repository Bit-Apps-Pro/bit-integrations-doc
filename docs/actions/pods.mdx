> **Note: Pods is available in the free and pro versions.**

***To integrate with Pods follow, the below steps.***


## Process of Authorization and Integration

1.	Search & Select **Pods** from the actions pages
 <Image src="actions/pods/search-and-select" alt="Pods" w="1020" h="350" />

2. **Integration Name:** Set the proper integration name. Default it's taken 'Pods'.
 <Image src="actions/pods/default-integration-name" alt="Pods" w="1020" h="350" />

3. After navigate to the integration page, select the ***Post Type*** you have created for the integration. For example, we have selected `Schools` as custom post type. 

 <Image src="actions/pods/select-post-type" alt="Pods" w="1020" h="350" />

4. Now, select **`Post Status`**, **`Author`** & **`Comment Status`** according to the requirement.
 <Image src="actions/pods/status-author-comment" alt="Pods" w="1020" h="350" />

5. For the field mapping option, you have to configure three types of mapping-
    - At first map the form fields with the general/default **post fields**.
    - Then map the form fields with the related **Pods fields**.
    - Lastly map the **file upload field**.
 <Image src="actions/pods/filed-mapping" alt="Pods" w="1020" h="350" />

6. **`Save`** all the integration settings. 
 <Image src="actions/pods/save-integration" alt="Pods" w="1020" h="350" />

7. We have an option in every integration **`Timeline`** where you can check integration success or error status.
 <Image src="actions/pods/pods-timeline" alt="Pods" w="1020" h="350" />
 <Image src="actions/pods/pods-log" alt="Pods" w="1020" h="350" />

8. Also, you can **`Edit`** **`Clone`** **`Delete`** the integration.

 <Image src="actions/pods/pods-option" alt="Pods" w="1020" h="350" />
