Authorization Process With Email

The function of an email notification is to notify the user that the form/task has been submitted successfully.

To set up the email notification feature follow the below steps-

Note: Mail is available on the free and pro version.

  1. Search & Select Mail from the actions pages
  1. Integration Name: Set the proper integration name Default it's taken ‚ÄúMail Integrations‚Äù.

Other fields you have to set are:

‚úî From: Set the email address from where you want to send mail to recipients.

‚úî To: Here you have to input the email address where you want to send the email notification. Also you can select the ‚ÄúForm Fields‚Äù to fetch the data directly from the frontend.

‚úî CC & BCC field: Then you have chosen the CC & BCC field if you want to send the notification to multiple email addresses.

‚úî Reply To: Set the email address where the user will send the reply message in response to your email notification.

‚úî Attachment: You can send an attachment if you have any file upload field on the form

‚úî Subject: You can manually enter the subject line and place dynamic fields in the subject line.

‚úî Template:

  • You can put dynamic field names email body

  • You can allow HTML and CSS for styling

  • You can copy and paste previous design template another editor

  • You can edit styling by the help of rich text editor

  1. After giving all the required information, click on Save. Now ready to send the email notification you created.

  2. We have an option in every integration "Timeline," where you can check integration success or error status.

info

At last, you need to make sure that the SMTP configuration is correct, to send the email without any spamming.
